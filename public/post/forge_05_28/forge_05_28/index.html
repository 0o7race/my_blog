深度学习驱动的全球天气预报方向研究综述笔记

本文在原有研究笔记的基础上进行了整理与润色，尽量保留既有模型分析、模块拆解、研究问题和比较结论，主要对结构、语句和排版进行优化，使其更适合作为一篇连续可读的方向综述。文中沿用原笔记中的“下垂直”表述，主要指将格点场解码为站点预测的过程。

1. 方向背景

1.1 任务定义

这一方向的核心任务，是利用深度学习方法，根据多种气象观测或再分析场数据，预测未来一段时间内的全球多层大气变量场。输入可以是历史天气场，例如 ERA5，也可以是卫星、探空、地面观测等多源观测数据。输出通常包括地表及不同气压层上的温度、风速、湿度、气压等物理量场；在某些任务中，还会进一步输出站点尺度的点位预测结果。

从输入形式来看，不同模型的假设差异较大。Aardvark 直接处理不规则观测数据，通过 convDeepSet 将观测投影到规则格点；GraphCast 和 GenCast 直接以 ERA5 等规则网格场为输入；Pangu-Weather 使用高分辨率、多层次的全球大气场；FuXi-S2S 则以过去若干天的全球场和时间特征为条件，进行更长时效的亚季节预报。

从输出形式来看，大多数模型最终都输出格点场。GraphCast、GenCast 和 Pangu-Weather 侧重全球大气场本身的预测；Aardvark 除了格点同化和预报，还会将结果进一步解码到站点层面；FuXi-S2S 则以每日 NetCDF 场为主要输出格式。

1.2 输入输出与评价指标

这一方向的常用评价指标主要包括：

  • RMSE、MAE：衡量连续场的整体误差；
  • ACC：衡量场型相关性，是天气预报中常见的结构性指标；
  • CRPS：在集合预报和概率预报中尤其重要，用于评估分布预测质量；
  • 极端事件命中率及相关指标：用于检验台风、强风、暴雨等高影响天气的表现。

不同模型关注的指标侧重点并不完全相同。确定性模型通常更看重 RMSE 和 ACC；概率模型则会进一步关注集合校准、spread-skill 一致性以及极端事件概率分布的合理性。

1.3 应用场景

深度学习驱动的全球天气预报主要面向以下场景：

  • 数值天气预报的后处理与误差订正；
  • 极端天气预警；
  • 快速多成员集合预报；
  • 气候风险评估与长期趋势分析；
  • 更低成本的全球天气模拟与业务试验系统。

从应用角度看，这一方向的价值并不只是“替代传统模式”，更重要的是提供一种更高效、更易扩展的天气预报新范式。

1.4 核心困难

全球天气场预测的难点主要集中在以下几个方面：

  1. 数据稀疏与数据形态不统一。 地面观测远少于格点数量，不规则观测与规则网格之间存在明显形态差异。
  2. 多变量耦合复杂。 温度、风场、湿度和气压等变量之间存在复杂的物理相互作用，单变量建模很难充分反映实际演化。
  3. 局地细节与全球结构之间存在张力。 一方面需要捕捉对流尺度和局地锋面，另一方面又要刻画行星尺度的全球传播。
  4. 多步预测误差累积明显。 自回归模型在长时效推理中容易产生漂移和偏差积累。
  5. 计算复杂度高。 以 0.25° 分辨率表示全球多层大气场，数据规模本身就非常庞大。
  6. 泛化能力仍有限。 模型在不同初始场、极端事件和分布外情形下的稳定性，仍是一个开放问题。

2. 模型谱系与主要技术路线

从技术路径上看，这一方向中的代表模型大致可以归为以下几类：

2.1 CNN / ConvCNP 路线：局地建模与观测同化

以 Aardvark 为代表。这类方法更强调从真实观测出发，通过卷积、深度集合映射和局地预测网络完成从观测到格点、再到站点的端到端建模。其重点不只是天气场外推，还包括同化、插值和下垂直等环节。

2.2 图神经网络路线：球面全局建模

以 GraphCast 为代表。这类方法将地球网格映射到多层球面 mesh 结构上，通过图消息传递建模全球大气场的传播与耦合。其优势在于天然适配球面几何，更适合建模全球尺度的相互作用。

2.3 概率生成路线：GNN 与扩散模型结合

以 GenCast 为代表。它在 GraphCast 的球面 mesh 框架基础上进一步引入扩散生成范式，使模型不再只输出单一路径，而是可以生成带不确定性的集合样本。其重点从确定性误差最小化，转向对未来分布的显式建模。

2.4 分层 Transformer 路线：局部窗口与多尺度融合

以 Pangu-Weather 为代表。该路线强调局部窗口注意力、三维结构建模和 U-Net 风格的多尺度融合，更适合高分辨率下的局地细节恢复和垂直结构建模。

2.5 长时效自回归 S2S 路线

以 FuXi-S2S 为代表。该方向更关注亚季节尺度的长期预报，强调在多天到数十天尺度上的滚动预测和时间条件控制。

2.6 模型之间的关系

若从研究脉络来理解：

  • GraphCast 可以看作全球确定性预报中的强基线；
  • GenCast 是在 GraphCast 基础上引入概率生成范式的扩展版本；
  • Pangu-Weather 与 GraphCast 都服务于全球中期预报，但更强调高分辨率和局地细节；
  • Aardvark 走的是不同路线，更重视多源观测同化与站点级任务；
  • FuXi-S2S 则把问题推进到了更长时效的 S2S 任务。

因此，这几个模型虽然都属于“深度学习驱动的全球天气预报”，但各自切入的问题并不完全相同。

4. 结构对比

4.1 输入与嵌入模块

  • Aardvark 使用 convDeepSet 将不规则观测映射到内部网格，并叠加地形、气候基线等辅助特征。
  • GraphCast / GenCast 将历史场与太阳辐射、地形等静态外力拼接后，作为格点节点特征输入。
  • Pangu-Weather 使用 PatchEmbedding 做三维卷积式嵌入，将不同高度层与地表变量统一到局部 token 表示中。
  • FuXi-S2S 的输入处理基本封装在 ONNX 管线内，从公开代码看主要是时间步和日期等条件控制。

从设计哲学上看，Aardvark 关注“如何把稀疏观测合理投影到场上”；GraphCast 和 GenCast 关注“如何从完整场出发做全局传播”；Pangu 更像是在高分辨率网格上做结构化压缩。

4.2 主干与编码器

  • Aardvark 的主干可以理解为局地卷积与 ViT 的混合网络；
  • GraphCast 的核心是 DeepTypedGraphNet，通过 mesh 节点上的消息传递建模全球传播；
  • GenCast 在 mesh 结构之上使用稀疏 Transformer 和扩散生成；
  • Pangu-Weather 则由多层 EarthSpecificBlock 组成，采用局部窗口 3D 注意力；
  • FuXi-S2S 的内部结构未公开，只能从推理流程推测其具备时序自回归能力。

4.3 Block 级设计

  • Aardvark 的基础单元以 3×3 卷积和可选 ViT 模块为主；
  • GraphCast 的“块”本质上是图消息传递层；
  • GenCast 的基本块更接近 Transformer 的多头注意力结构，但作用在球面 mesh 或 band 上；
  • Pangu 的基本块是局部窗口注意力与卷积混合；
  • FuXi-S2S 因代码缺失，无法进一步分解。

4.4 局部与全局建模

  • 局部建模更强：Aardvark、Pangu-Weather
  • 全局传播更自然：GraphCast、GenCast
  • 长时时序控制更明显：FuXi-S2S

GraphCast 的优势在于通过球面 mesh 避免规则经纬网格在高纬度区域的畸变问题；Pangu 的优势在于更擅长捕捉高分辨率局地结构；Aardvark 则更关注真实观测场景下的局地同化和站点任务。

4.5 多尺度建模

  • Pangu-Weather 有最明确的 U-Net 式多尺度结构；
  • GraphCast 的多尺度主要体现在不同细化级别的 mesh 上；
  • Aardvark 的多尺度更多体现在“观测 → 格点 → 站点”的任务链条中；
  • GenCast 在空间结构上继承 GraphCast，在目标空间上多了一层概率维度；
  • FuXi-S2S 更偏时间级联，而不是显式空间金字塔。

4.6 特征融合与解码

  • Aardvark 更多依赖简单拼接与分阶段解码；
  • GraphCast / GenCast 通过 mesh2grid_gnn 把 mesh 信息映射回规则网格；
  • Pangu 通过 skip connection 和上采样恢复细节；
  • FuXi-S2S 的解码细节未公开。

4.7 Head 与任务输出

  • Aardvark 的站点 Head 最特殊，直接面向下垂直任务；
  • GraphCast / GenCast 直接输出大气场；
  • Pangu 使用 PatchRecovery 恢复物理网格；
  • FuXi-S2S 则直接输出场文件。

5. 核心模块横向详解

5.1 输入嵌入：convDeepSet、grid2mesh 与 PatchEmbedding

这三类输入模块分别针对三种完全不同的问题：

  • Aardvark 的 convDeepSet 解决的是不规则观测到规则格点的投影问题；
  • GraphCast 的 grid2mesh_gnn 解决的是从规则网格到球面图结构的几何映射问题；
  • Pangu 的 PatchEmbedding 则解决的是高分辨率网格场的压缩与三维局部结构编码问题。

三者虽然都属于“输入嵌入”，但本质目标并不相同：一个偏同化，一个偏几何重表示，一个偏高分辨率 token 化。

5.2 Backbone / Encoder：局地、球面与三维三种范式

  • Aardvark 适合从复杂观测流出发，做较灵活的端到端系统；
  • GraphCast 更适合全球传播建模，尤其适合中期全球场预测；
  • GenCast 则在 GraphCast 基础上把确定性建模推进到概率生成；
  • Pangu 在高分辨率建模上更有优势，尤其适合需要保持局地结构的变量；
  • FuXi-S2S 虽然研究价值存在，但因训练结构不透明，其可分析性较弱。

如果从研究延展性来看，GraphCast 和 Aardvark 的价值往往更高，因为它们公开性更强、结构更清晰。

5.3 局部建模模块：卷积、注意力与潜在的 SSM

Aardvark 和 Pangu 中的卷积层负责提取局地特征，带有很强的归纳偏置；Pangu 的窗口注意力负责在较小三维邻域内建模精细结构；GraphCast 的局地关系更多是通过图边邻接来隐式表示。

目前这几个代表模型中，原生使用 SSM / Mamba 的还不多，因此这部分仍然存在明显的研究空间。例如，可以尝试用 SSM 替代一部分局部注意力或长时依赖模块，以降低计算复杂度。

5.4 多尺度模块：空间、几何与任务链三种“尺度”

“多尺度”在这些模型里并不是同一个概念：

  • Pangu 的多尺度 是分辨率金字塔；
  • GraphCast 的多尺度 是不同粒度的球面 mesh；
  • Aardvark 的多尺度 则更像任务链条上的多尺度：观测、格点、站点；
  • FuXi-S2S 更偏时间维级联，而非显式空间尺度。

这一点很重要，因为它决定了不同模型的“多尺度优势”并不能直接等价比较。

5.5 特征融合与 Decoder

Aardvark 主要使用简单拼接，易于实现但对源间冲突建模不足；GraphCast 的 mesh2grid_gnn 更结构化，能利用球面几何信息；Pangu 则采用 U-Net 风格的跳跃连接，兼顾高低层信息。

如果后续想做融合机制上的创新，交叉注意力、门控融合或结构化对齐模块都是值得尝试的方向。

5.6 任务 Head

Aardvark 的 ConvCNPWeatherOnToOff 是一个很有价值的模块，因为它说明“格点到站点”的下游任务完全可以设计成可学习 Head，而不是简单插值。这个思路事实上也可以迁移到 GraphCast 或 Pangu 上。

5.7 Normalization 与位置编码

Aardvark 和 GraphCast 都在损失层面显式考虑纬度权重；Pangu 在注意力中引入地球坐标相关偏置；GraphCast 通过图边属性编码地理关系；GenCast 还需要额外编码噪声层级。

从研究角度看，这说明全球天气预报中的“位置编码”并不是普通视觉任务中的可有可无组件，而是与球面几何和物理结构直接相关的关键先验。

7. 参数与训练策略对比

7.1 结构参数

  • Aardvark 的输入分辨率相对较低，但通道构成复杂，由观测源决定；
  • FuXi-S2S 的公开推理输入形状较固定，例如带时间和变量维度的多层场；
  • GraphCast 常见设置包含 37 个气压层，隐藏维度较高，消息传递步数通常有限但每步成本不低；
  • GenCast 在 GraphCast 基础上还需要额外的噪声条件与更深的生成结构；
  • Pangu-Weather 的输入分辨率与参数量都最大之一，属于重型高分辨率模型。

7.2 训练流程

  • Aardvark：常采用分阶段训练，再进行 E2E 微调；
  • GraphCast：依赖大规模 ERA5 训练，配合变量权重和归一化；
  • GenCast：在 GraphCast 风格结构之上增加噪声训练与采样流程；
  • Pangu：通常需要为不同时间步长或不同预报间隔训练多个模型；
  • FuXi-S2S：因缺少训练代码，难以做更细分析。

7.3 推理策略

  • Aardvark 可同时返回格点场与站点结果；
  • FuXi-S2S 通过 total_step 等参数控制时长和成员数；
  • GraphCast 支持较长 rollout；
  • GenCast 需要考虑采样步数和 ensemble size；
  • Pangu 在工程上会拼接不同时间步模型完成完整推理。

7.4 公平比较问题

这些模型在输入分辨率、参数量、训练策略和推理设置上差异很大，因此很难直接用单一指标判断孰优孰劣。例如：

  • Pangu 的高分辨率本身就会带来局地细节优势；
  • GenCast 的性能会受到采样步数和集合大小的强烈影响；
  • Aardvark 因使用真实观测输入，与 ERA5 驱动模型并不完全处于同一设定下。

因此，若要严谨比较，必须统一数据源、输入分辨率、训练预算、评估变量和后处理规则。

8. 数据流、Loss 与 Metric 对比

8.1 数据集与预处理

  • Aardvark 在 WeatherDatasetAssimilation 中返回字典式任务输入，包含同化、预测等多个子任务字段；
  • GraphCast / GenCast 更依赖 xarray.Dataset 形式的规则场输入；
  • Pangu 假设输入已完成归一化与张量整理；
  • FuXi-S2S 推理中主要做单位转换、气压维顺序处理和 SST 掩码等。

8.2 数据增强

天气场不同于普通图像，很多视觉增强方式并不适用。例如：

  • 不能随意水平翻转，否则会破坏地球物理意义；
  • 也不能做颜色扰动，因为每个通道都对应具体物理量。

因此，这一方向的增强通常较保守。Aardvark 里可能通过随机预测时效增强时间鲁棒性；GenCast 的噪声训练本身也可视为一种“扰动式增强”。

8.3 Loss 设计

  • Aardvark：经纬加权、压力层加权、变量加权的 RMSE 类损失；
  • GraphCast：纬度和层级加权的 MSE；
  • GenCast：在 GraphCast 基础上叠加噪声层级去噪目标；
  • Pangu：以 MAE 为主，并对地表分量额外加权；
  • FuXi-S2S：未公开，但大概率是 MSE 或 MAE 类损失。

8.4 评价指标

  • 确定性模型：RMSE、ACC 为主；
  • 概率模型：CRPS、spread-skill、可靠性图等更重要；
  • Aardvark：还额外关注站点尺度误差；
  • Pangu：在特定层变量上常展示 Z500、T850 等指标。

8.5 Loss 与 Metric 是否一致

这也是一个值得注意的问题。GraphCast 和 Aardvark 的损失函数与 RMSE 类指标相对一致；Pangu 的 MAE 与 RMSE 有一定偏差；GenCast 的去噪 MSE 与最终 CRPS 更是存在代理目标与最终目标之间的 gap。这意味着训练时优化得好的模型，未必在所有业务指标上都同样最优。

9.1 细节保留

Pangu-Weather 在高分辨率细节保留方面最突出，因为其 patch 化比例较保守，又通过 skip connection 把高层细节带回输出。Aardvark 在格点预测上也具备较强局地能力，但格点到站点的下垂直过程仍可能损失一部分局地结构。GraphCast 与 GenCast 更偏整体场型，局地细节往往不如 Pangu 鲜明。

9.2 全局建模

GraphCast 与 GenCast 几乎是“先全局、后输出”的代表；Pangu 则是“先局地，再逐层扩展到全局”；Aardvark 介于两者之间；FuXi-S2S 因实现不公开，不便判断。

9.3 多尺度融合

Pangu 的多尺度最显式；GraphCast 的多尺度带几何意味；Aardvark 的多尺度更偏任务链；GenCast 主要在目标空间加入概率维度，而不是额外的空间金字塔。

9.4 计算效率

  • 最高：FuXi-S2S（从公开推理流程看）
  • 较高：Pangu-Weather、GraphCast
  • 偏低：Aardvark
  • 最低：GenCast

当然，这种比较只是在公开实现层面的大致印象，具体仍取决于输入分辨率、模型大小和部署方式。

9.5 信息瓶颈

  • Aardvark：观测到格点、格点到站点都可能造成信息损失；
  • GraphCast：grid 到 mesh 压缩后，局地高频细节可能不足；
  • GenCast：噪声—去噪过程对条件质量和采样策略依赖大；
  • Pangu：patch 化和低分辨率阶段可能损失超局部结构；
  • FuXi-S2S：主要瓶颈是结构不可见，难以定量分析。

10. 贡献与消融实验判断

10.1 各模型的主要贡献

  • Aardvark
    贡献在于多源观测同化、端到端格点到站点流程，以及适合业务任务的完整系统设计。

  • FuXi-S2S
    贡献主要在于更长时效 S2S 场景及时间条件控制思路，但由于训练结构不透明，研究价值更多体现在任务设定而非结构公开。

  • GraphCast
    贡献在于球面图神经网络范式的成熟化与工程化，是全球天气预测领域的重要公开基线。

  • GenCast
    贡献在于把扩散生成引入全球天气预报，使概率预报具备了更强的表现能力。

  • Pangu-Weather
    贡献在于 3D 地球局部注意力与高分辨率多尺度结构，为高精度局地建模提供了重要路线。

10.2 这些贡献中哪些更可信

GraphCast 的结构创新最稳固，因为其设计、代码和实验都较完整；Aardvark 的工程意义很高，但结构收益与训练 recipe 之间需要更严格地区分；GenCast 的创新明确，但必须把采样步数、噪声调度等因素单独控制；Pangu 的结构很有启发性，但受限于完整开源程度，许多判断仍需要依赖论文与复现。

10.3 主要混淆因素

在比较模型时，以下因素很容易掩盖真正的结构差异：

  • 训练数据规模和数据清洗策略；
  • 是否使用再分析初值；
  • 损失函数的权重设计；
  • 采样步数、ensemble size；
  • 是否使用更高分辨率输入；
  • 是否做了多模型时间步拼接。

10.4 可行的跨模型消融建议

  • 将 Aardvark 的 convDeepSet 用作 GraphCast 的观测编码器，测试观测驱动模式是否可行；
  • 将 Aardvark 的站点解码器 接到 GraphCast 或 Pangu 输出后，比较站点预报性能；
  • 在 GraphCast 中加入 Pangu 风格的垂直编码或局部窗口分支，比较局地细节是否改善；
  • 将 GraphCast 的加权损失迁移到 Aardvark，检验极端变量权重设计对不同结构的影响；
  • 在 GenCast 中改变噪声 schedule 与采样步数，分离“结构贡献”和“推理技巧贡献”。

11. 方向级局限性

从整个方向来看，目前仍有一些共性瓶颈：

  1. 对再分析场依赖较强。 除 Aardvark 外，大多数模型仍默认已拥有较高质量的初始全球场。
  2. 极端事件学习不足。 平均误差导向的损失函数仍然占主流，对高影响天气的关注不够。
  3. 长时效误差积累明显。 结构上的稳定性增强仍不够，物理反馈也往往被弱化。
  4. 跨季节、跨年份与跨数据源泛化不足。 训练分布外的行为仍不够明确。
  5. 复杂地形与海陆交界带仍是难点。 地形编码虽然已被加入，但表达仍不充分。
  6. 可解释性不足。 尤其是生成模型和大型 Transformer，很难直接解释其物理依据。

12. 研究 Idea

12.1 保守方向：极端事件加权损失

来源： GraphCast、Aardvark
核心假设： 如果在损失函数中提高极端事件样本的权重，模型对极端天气的刻画会明显提升。
方法： 在现有 weighted MSE 或 weighted RMSE 上，对超过某个分位阈值的区域增加额外权重。
可能效果： 提高强风、暴雨、高温等极端事件预测性能。
验证方式： 比较极端分位样本上的误差、命中率与常规 RMSE 的变化。
风险： 可能牺牲整体平均误差，需要平衡权重。

12.2 保守方向：通用站点下垂直模块

来源： Aardvark
核心假设： 一个可插拔的站点解码器可以提升任何格点模型的站点级预测表现。
方法： 保留 GraphCast 或 Pangu 的格点输出，在其后接入 ConvCNPWeatherOnToOff 风格的站点 Head。
效果预期： 改善站点温度、风速等任务的 MAE / CRPS。
风险： 如果格点场本身偏差过大，解码器收益会有限。

12.3 中等方向：垂直感知图网络

来源： Pangu 的 3D 思路 + GraphCast
核心假设： 在 GraphCast 的节点特征中显式加入垂直剖面编码，可以提高高空变量和跨层耦合变量的精度。
方法： 在 _inputs_to_grid_node_features() 中增加面向垂直层的 MLP 或 1D Conv 编码。
验证方式： 比较加入与不加入垂直编码时，不同气压层的 RMSE / ACC。
风险： 参数增加但增益不一定明显，且可能与现有变量权重机制冲突。

12.4 中等方向：局地—全局混合处理器

来源： GraphCast + Pangu
核心假设： 同时建模局部窗口和球面 mesh，能够兼顾局地细节和全球传播。
方法： 在 GraphCast 中增加一条原始网格上的局部窗口注意力分支，与 mesh GNN 并行处理，再在后期做门控融合。
预期效果： 局地结构和全球波动同时改进。
风险： 架构复杂度和训练难度都会明显上升。

12.5 激进方向：掩码预训练

来源： 自监督学习
核心假设： 在天气场上进行随机掩码重建，有助于模型学习变量间和层间耦合，提高数据效率和鲁棒性。
方法： 在 GraphCast 或 Pangu 上加入掩码重建预训练，再进行下游微调。
验证方式： 比较是否能在小样本或分布偏移数据上获得更强泛化。
风险： 预训练任务若设计不当，可能更偏向学习平滑趋势而损失高频结构。

12.6 激进方向：跨数据源鲁棒模型

来源： 现有模型对 ERA5 依赖较强
核心假设： 如果训练中混合 ERA5、HRES、JRA-55 等多种数据源，并显式加入 source embedding，模型可能学到更稳定的跨源表示。
方法： 扩展数据管线，加入数据源标签嵌入，并进行联合训练。
预期效果： 提升跨数据源迁移能力。
风险： 数据准备和对齐成本很高，不同源之间的统计偏差也会增加训练难度。

13. 推荐研究路线

13.1 入门学习路径

若作为入门，最推荐先从 GraphCast 开始。原因很简单：其结构清晰、公开程度高，而且恰好处于“全局天气场预测基线”的核心位置。读懂 grid → mesh → mesh → grid 之后，再去看 Pangu 和 GenCast，会更容易把握差异。

13.2 Baseline 选择建议

  • 若偏好 全球场结构建模，建议以 GraphCast 为 baseline；
  • 若偏好 多源观测融合和站点任务，建议以 Aardvark 为 baseline；
  • 若偏好 概率预报，建议在 GraphCast 基础上延伸到 GenCast；
  • 若偏好 高分辨率局地细节建模，则 Pangu-Weather 更值得深入。

13.3 适合起步的研究问题

  • 观测驱动而非再分析驱动的全球天气预报；
  • 多尺度球面混合结构；
  • 极端天气损失函数设计；
  • 可插拔站点下垂直模块；
  • 掩码预训练与自监督学习在天气场上的可行性。

14. 研究级问题与标准答案（基础篇）

  1. 问：这些模型是否解决同一个问题？
    答： 不完全相同。Aardvark 更偏观测同化和站点预报，GraphCast、GenCast 与 Pangu-Weather 更偏全球大气场预报，FuXi-S2S 则面向更长时效的亚季节场预测。任务设定不同，导致输入输出和指标也并不统一。

  2. 问：为什么天气场预测不能简单当作普通图像回归任务？
    答： 因为天气场不只是二维图像，而是包含多变量、多气压层、球面几何与长时依赖的复杂物理系统。再加上观测稀疏、物理约束和多步预测误差累积，这些都远超普通图像回归的假设。

  3. 问：哪些模型使用再分析场，哪些直接使用观测？
    答： GraphCast、GenCast、Pangu-Weather 和 FuXi-S2S 都以规则网格的再分析场或类似输入为主；Aardvark 则直接从多源观测出发。这会直接影响比较的公平性，因为输入信息量本来就不同。

  4. 问：长时效天气预测最难的地方是什么？
    答： 核心问题是误差累积。多步自回归每前进一步都会引入新偏差，之后这些偏差还会继续被当作输入放大。

  5. 问：为什么上层变量预测好，不代表降水或站点温度也好？
    答： 因为上层变量通常更平滑、更偏大尺度，而降水和站点变量往往局地性强、非线性更重、极端值更多。优化平均误差的模型更容易在前者上表现好，在后者上则未必。

  6. 问：Aardvark、FuXi-S2S、GraphCast、GenCast、Pangu-Weather 的主要技术路线分别是什么？
    答： Aardvark 偏向 ConvCNP 与多源同化，FuXi-S2S 偏长时效自回归，GraphCast 是球面 GNN，GenCast 是 GraphCast 上的扩散概率化版本，Pangu-Weather 是分层 3D 局部注意力 Transformer。

  7. 问：GraphCast 和 Pangu-Weather 在空间建模上有什么根本差异？
    答： GraphCast 先把场映射到球面 mesh 上，再通过 GNN 做全球传播；Pangu 则保留规则网格表示，在局部窗口中做 3D 注意力，并通过层级堆叠扩展视野。

  8. 问：为什么不能把 GenCast 简单理解为“GraphCast 加噪声”？
    答： 因为 GenCast 不只是往输入里加噪，而是把任务目标改成了“从噪声恢复未来场”的生成式学习问题。它学习的是条件分布，不再只是单一回归。

  9. 问：Aardvark 为什么被称作端到端天气系统？
    答： 因为它不只包含一个 backbone，而是把观测同化、格点预测和站点下垂直串成了一整条流水线。

  10. 问：FuXi-S2S 从公开代码里能确认什么？
    答： 可以确认它做了基于时间特征的长时自回归预测，并用 ONNX 执行推理。但其内部主干结构、损失函数和训练方式仍不可见。

  11. 问：GraphCast 的“grid → mesh → mesh → grid”具体是什么意思？
    答： 先把规则经纬网格编码成球面 mesh 节点特征，再在 mesh 上做多轮消息传递，最后再把更新后的 mesh 特征投影回规则网格输出预测。

  12. 问：graph_mesh_connectivity.py 和 icosahedral_mesh.py 的作用是什么？
    答： 它们负责构建球面网格及 grid 和 mesh 之间的连接关系，是 GraphCast 几何结构的基础。

  13. 问：为什么 DeepTypedGraphNet 比普通 CNN 更适合全球传播？
    答： 因为它不受规则网格卷积核的限制，能够通过球面图结构更自然地传播远距离信息，更符合全球尺度场传播的需求。

  14. 问：GenCast 训练时为什么要先加噪再去噪？
    答： 这是扩散模型的基本范式。通过在不同噪声水平下恢复真实场，模型可以学习未来分布，而不是只输出一个平均解。

  15. 问：FourierFeaturesMLP 为什么要对噪声层级做频率编码？
    答： 因为不同噪声层级对应不同的去噪任务难度，频率编码能让模型更容易感知并区分这些层级。

  16. 问：采样步数、噪声调度和 stochastic churn 对 GenCast 有什么影响？
    答： 这些都会显著影响最终样本质量、推理速度和比较公平性。更多步通常更准，但更慢；噪声调度设计不当会直接损害生成质量。

  17. 问：Pangu 的 PatchEmbedding 为什么对高空变量用 Conv3D、对地表变量用 Conv2D？
    答： 因为高空变量天然包含垂直层维度，而地表变量没有。这样做可以更自然地把垂直结构纳入建模。

  18. 问：EarthAttention3D 与普通 2D 窗口注意力有何区别？
    答： 最大区别在于它同时建模水平和垂直维度，并且考虑地球几何相关偏置，更适合大气场三维结构。

  19. 问：Pangu 为什么采用多模型迭代推理，而不是一个统一模型滚动到底？
    答： 因为不同时间步长的模型各自更擅长特定时距的预测。这样做能减轻长时自回归误差，但代价是工程复杂度更高。

  20. 问：convDeepSet、grid2mesh_gnn 和 PatchEmbedding 都是输入模块，它们解决的是同一个问题吗？
    答： 不是。convDeepSet 解决观测对齐，grid2mesh_gnn 解决几何映射，PatchEmbedding 解决高分辨率压缩与局部表示学习。

  21. 问：GraphCast 的 mesh 消息传递与 Pangu 的 3D 窗口注意力，哪个更偏全局，哪个更偏局部？
    答： GraphCast 更偏全局，Pangu 更偏局部。前者依靠球面传播，后者依靠局部窗口逐层扩展视野。

  22. 问：Aardvark 直接用 cat 融合多源观测有什么利弊？
    答： 好处是实现简单、计算便宜；不足是缺少显式的源间权重分配与冲突处理机制。

  23. 问：GraphCast 的 mesh2grid_gnn 与 Pangu 的 PatchRecovery 有什么本质区别？
    答： 前者是基于球面图结构的解码，后者是规则网格上的卷积式恢复。两者面对的几何约束不同。

  24. 问：GenCast 的稀疏 Transformer 去噪器与 GraphCast 的 GNN 在目标上有什么不同？
    答： GraphCast 做确定性回归；GenCast 做条件去噪生成。前者追求唯一最优输出，后者追求样本分布。

  25. 问：Pangu 的多尺度与 GraphCast 的多 mesh，为什么不能简单视为一回事？
    答： 因为前者是分辨率金字塔，后者是球面几何粒度变化。它们的“尺度”含义并不一样。

  26. 问：Aardvark 的 PressureWeightedRmseLoss 与 GraphCast 的加权 MSE 有什么异同？
    答： 相同点是都在考虑纬度与层级差异；不同点是 GraphCast 更强调多变量统一权重，Aardvark 更贴近其同化与下垂直任务设计。

  27. 问：Pangu 的 Earth-specific bias 与 GraphCast 的空间边特征，归纳偏置有什么不同？
    答： Pangu 的偏置直接作用在注意力分布上，GraphCast 的边特征则作用在图消息传递过程中。一个偏注意力，一个偏拓扑传播。

  28. 问：如果要把 Aardvark 改成概率模型，最关键要改哪里？
    答： 需要修改输出 Head 和损失函数，让模型输出分布参数或样本，而不再只是单一均值。

  29. 问：GraphCast 的强表现能否只靠训练 recipe 解释，而不需要结构创新？
    答： 不能。训练 recipe 很重要，但球面 GNN 的结构优势依然是核心原因。两者不能简单割裂。

  30. 问：这个方向里哪些设计空间已经比较成熟，哪些仍明显不足？
    答： 球面 GNN、窗口注意力和加权误差设计已相对成熟；多源观测融合、极端事件优化、跨源泛化和可解释性仍明显不足。

15. 研究级问题与标准答案（延伸篇）

  1. 问：Aardvark、GraphCast 和 Pangu 在局地与全球信息处理上的差异可以如何概括？
    答： Aardvark 更像“局地为主、全局补充”，GraphCast 更像“全局优先”，Pangu 则是“局地起步、逐层扩展”。

  2. 问：如果把 FuXi-S2S 的 step 和 doy 条件加入 GraphCast，理论上有什么潜在收益？
    答： 可能改善长时效预测和季节性适应能力，让模型更明确感知预测时距与年内背景。

  3. 问：为什么公平比较时必须控制参数量和 FLOPs？
    答： 因为规模更大的模型天然有更强表达力。若不控制这些因素，比较结果很容易被模型规模掩盖。

  4. 问：Aardvark 使用 torch.nanmean 与严格 mask 策略有什么区别？
    答： nanmean 是在计算时被动跳过缺失值，mask 则是主动指定哪些区域参与训练或评估。后者更可控，前者更简单。

  5. 问：为什么多模型比较时必须统一 evaluation protocol？
    答： 因为不同 mask、不同变量范围和不同加权方式都会直接影响结果。若 protocol 不统一，指标对比就不可靠。

  6. 问：3D 窗口注意力相对于 2D 窗口注意力的代价是什么？
    答： 计算更贵、参数更多、对高度维设置更敏感，也更依赖输入层结构的一致性。

  7. 问：如果 GraphCast 改成每一步都单独归一化，而不是块级归一化，会怎样？
    答： 可能更接近普通单步自回归，但长期稳定性未必更好。这本身就是一个值得做的消融实验。

  8. 问：如何更合理地评估模型在极端天气中的表现？
    答： 不能只看 RMSE，需要增加极端分位误差、事件命中率、分类指标以及概率可靠性分析。

  9. 问：GraphCast 的 skipna=False 与 Aardvark 的 torch.nanmean 在缺失值处理上有何不同？
    答： 前者要求输入基本无缺失，否则损失会直接出问题；后者则允许缺失值存在，并在统计时跳过它们。

  10. 问：如果基于这些模型继续做研究，最值得优先关注什么？
    答： 最值得关注的仍是多源观测融合、极端天气优化、概率预报、效率提升和跨尺度泛化这几个方向。它们既有现实价值，也还有明显的结构和方法空间。
